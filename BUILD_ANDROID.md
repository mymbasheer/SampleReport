Download from **https://developer.android.com/studio**

During installation, make sure these are checked:
- вњ… Android SDK
- вњ… Android SDK Platform

After installing, set the environment variable:

**Windows** (run in Command Prompt as Admin):

```bash

export ANDROID_HOME=$HOME/Android/Sdk               # Linux
export PATH=$PATH:$ANDROID_HOME/platform-tools
```

Then restart your terminal.

### Step 1 вЂ” Install project dependencies

### Step 2 вЂ” Generate the native Android project

## рџ”‘ Signed vs Unsigned APK

The build above creates an **unsigned release APK** вЂ” this works fine for direct
installation. You only need signing if you want to upload to the Play Store.

To create a **signed APK**:

### 1. Generate a keystore (run once)
```bash
# Mac/Linux:
bash scripts/generate-keystore.sh

# Windows:
scripts\generate-keystore.bat
```

### 2. Add keystore details to `android/gradle.properties`
```properties
MYAPP_RELEASE_STORE_FILE=../../bizledger-release.keystore
MYAPP_RELEASE_KEY_ALIAS=bizledger
MYAPP_RELEASE_STORE_PASSWORD=your_password
MYAPP_RELEASE_KEY_PASSWORD=your_password
```

### 3. Update `android/app/build.gradle`
See `android-build-config/app.build.gradle.template` for exactly what to add.

### 4. Build signed APK
```bash
npm run build:android
```

## рџђ› Troubleshooting

| Error | Fix |
|-------|-----|
| `ANDROID_HOME not set` | Set the environment variable (see Step 3 above) |
| `SDK location not found` | Open Android Studio в†’ SDK Manager в†’ note the SDK path |
| `java: command not found` | Install Java 17 and restart terminal |
| `Could not resolve com.android...` | Check internet connection вЂ” Gradle downloads dependencies |
| `Execution failed for task ':app:mergeReleaseResources'` | Run `cd android && ./gradlew clean` then build again |
| Build hangs forever | Add `org.gradle.daemon=false` to `android/gradle.properties` |
